## 1. 시스템 개요

GV_TM은 전화 상담(TM) 업무를 통합 관리하는 시스템으로 세 가지 구성요소로 이루어져 있습니다.

```
┌─────────────────────────────────────────────┐
│              GV_TM 시스템 구성               │
│                                             │
│  [tm_web]  ─────────┐                       │
│  관리자 웹           │                       │
│                     ▼                       │
│               [Backend API]                 │
│               Django 서버                    │
│                     ▲                       │
│  [tm_mob]  ─────────┘                       │
│  상담원 모바일앱                               │
└─────────────────────────────────────────────┘
```

### 역할별 접근 권한

| 역할 | 설명 | 접근 가능 시스템 |
|------|------|----------------|
| **ADMIN** | 슈퍼 관리자 | TM_Web (모든 기능) |
| **MANAGER** | 팀장/매니저 | TM_Web (관리 기능) |
| **AGENT** | 상담원 | TM_Mob (상담 기능) |

### 핵심 업무 흐름

```
1. 관리자: 고객 DB 엑셀 업로드
      ↓
2. 관리자: 상담원에게 고객 배정 (자동/수동)
      ↓
3. 상담원: 모바일앱으로 고객 전화 & 기록 작성
      ↓
4. 관리자: 성공 건 → 2차 영업팀 배정
      ↓
5. 관리자: 정산 확인 및 지급 처리
```

## 2. 시스템 관리자 가이드

> 서버를 직접 운영하는 인프라/개발 담당자를 위한 가이드입니다.

### 2-1. 시스템 요구사항

| 항목 | 스펙 |
|------|------|
| OS | Linux (Ubuntu 권장) / macOS |
| Docker | 20.10 이상 |
| Docker Compose | 2.0 이상 |
| 포트 | 80, 8000, 5432, 6379 |

### 2-2. 초기 설치 및 실행

**1단계: 환경변수 설정**

```bash
# 프로젝트 루트에 .env 파일 생성
cp .env.example .env
```

`.env` 파일 필수 항목:

```env
# Django
SECRET_KEY=your-secret-key-here
DEBUG=False
ALLOWED_HOSTS=your-domain.com,localhost

# Database
POSTGRES_DB=tm_db
POSTGRES_USER=tm_user
POSTGRES_PASSWORD=your-db-password
DB_HOST=db
DB_PORT=5432

# Redis
REDIS_URL=redis://redis:6379/0

# JWT
JWT_SECRET_KEY=your-jwt-secret

# AWS S3 (녹취 파일 저장용, 선택사항)
AWS_ACCESS_KEY_ID=your-aws-key
AWS_SECRET_ACCESS_KEY=your-aws-secret
AWS_STORAGE_BUCKET_NAME=your-bucket-name
AWS_S3_REGION_NAME=ap-northeast-2
```

**2단계: 컨테이너 실행**

```bash
# 전체 서비스 시작
docker-compose up -d

# 실행 상태 확인
docker-compose ps
```

**3단계: 데이터베이스 초기화**

```bash
# 마이그레이션 실행
docker-compose exec backend python manage.py migrate

# 슈퍼유저 생성 (최초 1회)
docker-compose exec backend python manage.py createsuperuser
```

**4단계: Django Admin 접속**

```
http://your-domain.com/admin/
```
- 생성한 슈퍼유저로 로그인
- MemberLevel(회원등급) 먼저 생성 필요

### 2-3. MemberLevel(회원 등급) 설정

Django Admin → MemberLevel → 등급 추가:

| level_id | level_name | 용도 |
|----------|------------|------|
| 1 | admin | 최고 관리자 |
| 2 | manager | 팀장/매니저 |
| 3 | agent | 일반 상담원 |

### 2-4. 첫 번째 관리자 계정 생성

1. Django Admin → Accounts → 계정 추가
2. 또는 TM_Web 회원가입(`/signup`) 후 Admin에서 등급 설정

### 2-5. 컨테이너 구성

| 컨테이너 | 역할 | 포트 |
|---------|------|------|
| `backend` | Django API 서버 | 8000 |
| `db` | PostgreSQL | 5432 |
| `redis` | 캐시 / 메시지큐 | 6379 |
| `celery` | 비동기 작업 처리 | - |

### 2-6. 주요 관리 명령어

```bash
# 로그 확인
docker-compose logs -f backend
docker-compose logs -f celery

# 컨테이너 재시작
docker-compose restart backend

# DB 백업
docker-compose exec db pg_dump -U tm_user tm_db > backup.sql

# DB 복원
docker-compose exec -T db psql -U tm_user tm_db < backup.sql

# Celery 작업 상태 확인
docker-compose exec backend celery -A config inspect active
```

### 2-7. 업데이트 배포

```bash
# 코드 업데이트
git pull origin main

# 컨테이너 재빌드 및 시작
docker-compose up -d --build

# 마이그레이션 확인
docker-compose exec backend python manage.py migrate
```

### 2-8. 장애 대응

| 증상 | 원인 | 조치 |
|------|------|------|
| 엑셀 업로드 후 반영 안됨 | Celery 미작동 | `docker-compose restart celery` |
| 로그인 안됨 | JWT 시크릿 키 불일치 | `.env` 확인 후 재시작 |
| DB 연결 실패 | PostgreSQL 미기동 | `docker-compose restart db` |
| 정산 계산 오류 | SettlementPrice 미설정 | Django Admin에서 단가 설정 |

## 3. TM_Web 관리자 가이드

> TM_Web(웹 브라우저)을 통해 시스템을 운영하는 관리자/매니저를 위한 가이드입니다.

### 3-1. 로그인

1. 브라우저에서 TM_Web 주소 접속
2. 이메일과 비밀번호 입력 후 로그인
3. ADMIN 또는 MANAGER 권한이 있어야 접속 가능

### 3-2. 대시보드

로그인 후 메인 화면에서 전체 현황을 확인할 수 있습니다.

**주요 KPI 카드:**
- 오늘 총 콜 수
- 오늘 성공 건수
- 성공률 (%)
- 현재 활성 상담원 수

**차트:**
- 일별/주별 콜 추이
- 상담원별 성과 현황
- 상태별 배정 건수 분포

---

### 3-3. 상담원 관리 (`/dashboard/agents`)

#### 상담원 등록

1. **전제조건**: 상담원이 TM_Mob 앱에서 회원가입을 완료해야 함
2. `상담원 등록` 버튼 클릭
3. 후보자 목록에서 회원가입된 사용자 선택
4. 다음 정보 입력:

| 항목 | 설명 | 필수 |
|------|------|------|
| 사용자 | 회원가입된 계정 선택 | 필수 |
| 역할 | ADMIN / MANAGER / AGENT | 필수 |
| 팀 | SALES_TM(1차팀) / SALES_MAIN(2차팀) | 필수 |
| 담당 전화번호 | 상담에 사용할 발신 번호 | 선택 |
| 일일 배정 한도 | 하루 최대 배정 고객 수 | 선택 |
| 자동배정 여부 | 자동배정 시스템 포함 여부 | 선택 |

5. 등록 완료 시 **사번(TM-XXXXXX)** 자동 생성

#### 상담원 수정

- 상담원 목록에서 해당 상담원 클릭 → 정보 수정
- 상태 변경 가능: ONLINE / OFFLINE / BREAK / BUSY

#### 상담원 퇴사 처리

- 상담원 목록에서 `퇴사 처리` 버튼 클릭
- **주의**: 퇴사 처리 시 해당 상담원에게 배정된 모든 고객이 **미배정(NEW) 상태로 자동 회수**됨

---

### 3-4. 고객 DB 관리 (`/dashboard/customers`)

#### 엑셀 업로드

1. `엑셀 업로드` 버튼 클릭
2. 준비된 엑셀 파일(.xlsx) 선택
3. 업로드 후 **백그라운드에서 자동 처리** (대용량 가능)

**엑셀 파일 규격:**

| 필드 | 인식 가능한 헤더명 | 필수 |
|------|-----------------|------|
| 전화번호 | `전화번호`, `휴대폰`, `연락처`, `Tel`, `Phone` | **필수** |
| 이름/상호 | `이름`, `회사명`, `사찰명`, `상호명`, `대표자`, `성명` | 선택 |
| 업종 | `분야1`, `업종`, `업태` | 선택 |
| 세부업종 | `분야2`, `주생산품`, `종목` | 선택 |
| 세부분류 | `분야3` | 선택 |
| 시도 | `지역1`, `시도` | 선택 |
| 시군구 | `지역2`, `시군구` | 선택 |

> **참고**: 전화번호가 없는 행은 자동으로 건너뜁니다.
> 이미 존재하는 전화번호는 중복 무시됩니다.

#### 고객 목록 조회 및 필터

- **검색**: 이름, 전화번호, 지역으로 검색
- **필터**: 업종(분야1/2/3), 지역(시도/시군구)으로 필터링
- **상태별 탭**:
  - `1차 고객`: 전체 배정 대상 고객
  - `성공 고객`: 1차 성공 완료 건 (`/dashboard/customers-success`)
  - `재활용 후보`: 거절/결번 건 중 재활용 가능 건 (`/dashboard/customers-recycle`)

#### 고객 배정 (수동)

1. 고객 목록에서 배정할 고객 체크박스 선택
2. `배정` 버튼 클릭
3. 배정할 상담원 선택
4. 확인 → 해당 상담원 앱에 즉시 반영

#### 고객 배정 취소 / 삭제

- 배정된 고객 선택 → `배정 취소`: 미배정(NEW) 상태로 복귀
- 고객 선택 → `삭제`: 영구 삭제 (주의!)

#### DB 초기화

- 고객 관리 페이지 → `DB 초기화` 버튼
- **경고**: 모든 고객 데이터와 배정 기록이 영구 삭제됩니다.

---

### 3-5. 배정 관리

#### 자동 배정 실행

1. 상담원 관리 → `자동 배정 실행` 버튼
2. 또는 배정 관리 → `자동 배정`
3. 자동배정 대상 상담원의 `일일 배정 한도`에 맞게 미배정 고객 자동 분배

#### 배정 신청 처리 (`SalesPullRequest`)

상담원이 모바일 앱에서 "더 많은 고객 배정 신청"을 하면:

1. 배정 관리 → `배정 신청 목록` 확인
2. 신청 내용 검토 (신청 건수, 상담원 정보)
3. `승인` 또는 `거절` 처리
4. 승인 시 해당 수량만큼 고객 자동 배정

#### 2차 배정

1차 TM 성공 건을 2차 영업팀에 배정:

1. 성공 고객 목록에서 해당 고객 선택
2. `2차 배정` 버튼 클릭
3. SALES_MAIN 팀의 상담원 선택
4. 배정 완료

#### 배정 이력 조회 (`/dashboard/assignment-history`)

- 기간별 배정/성공/거절 이력 조회
- 상담원별 필터링
- 엑셀 export 가능

---

### 3-6. 성과 관리 (`/dashboard/performance`)

각 상담원의 성과를 카드/테이블 형태로 확인:

| 항목 | 설명 |
|------|------|
| 총 콜 수 | 전체 통화 기록 수 |
| 성공 건수 | SUCCESS 상태 건수 |
| 성공률 | 성공/전체 × 100 |
| 거절 건수 | REJECT 상태 건수 |
| 결번 건수 | INVALID 상태 건수 |
| 부재 건수 | ABSENCE 상태 건수 |

---

### 3-7. 정산 관리 (`/dashboard/settlement`)

#### 정산 현황 조회

- 기간 선택: 일별 / 주별 / 월별 / 직접 범위 설정
- 상담원별 정산 금액 확인

**정산 단가 (기본값):**

| 결과 | 단가 |
|------|------|
| 성공 (SUCCESS) | 2,000원 |
| 거절 (REJECT) | 700원 |
| 결번 (INVALID) | 0원 |

> 단가 변경은 시스템 관리자에게 요청 (Django Admin → SettlementPrice)

#### 정산 처리

1. 정산 관리 → 해당 기간/상담원 정산 건 클릭
2. 금액 확인 및 필요 시 수정
3. 상태 변경:
   - `PENDING` (미처리) → `REVIEW` (검토중) → `PAID` (지급완료)
4. 엑셀 export → 지급 처리

3. `상단 고정` 옵션: 중요 공지는 항상 최상단에 표시
4. 등록 → 상담원 앱에서 즉시 확인 가능

## 4. TM_Mob 상담원 가이드

> 스마트폰의 TM_Mob 앱을 사용하는 상담원을 위한 가이드입니다.

### 4-1. 앱 설치

- **Android**: 관리자로부터 APK 파일 수령 후 설치
  또는 Google Play Store에서 설치 (출시 시)
- **iOS**: App Store에서 설치 (출시 시)

### 4-2. 회원가입

1. 앱 실행 → `회원가입` 선택
2. 정보 입력:

| 항목 | 설명 |
|------|------|
| 이메일 | 로그인에 사용할 이메일 |
| 비밀번호 | 8자 이상 |
| 이름 | 실명 |
| 전화번호 | 본인 연락처 |

3. 회원가입 완료
4. **관리자에게 연락** → 상담원으로 등록 요청
5. 관리자가 등록 완료 후 앱 사용 가능

> **중요**: 회원가입만으로는 앱을 사용할 수 없습니다.
> 반드시 관리자가 상담원으로 등록해야 합니다.

### 4-3. 로그인

1. 앱 실행
2. 이메일 / 비밀번호 입력
3. `로그인` 버튼 탭

### 4-4. 배정 고객 확인 (`내 고객` 탭)

로그인 후 배정된 고객 목록이 자동으로 표시됩니다.

**고객 카드에서 확인 가능한 정보:**
- 고객명 (또는 전화번호)
- 전화번호
- 지역 / 업종
- 현재 상태

**상태 종류:**

| 상태 | 설명 |
|------|------|
| NEW | 새로 배정됨, 아직 연락 안함 |
| ASSIGNED | 배정됨 |
| TRYING | 통화 시도 중 (부재 등) |
| CALLBACK | 콜백 예정 |
| SUCCESS | 성공 |
| REJECT | 거절 |
| INVALID | 결번 |
| HOLD | 보류 |

### 4-5. 전화 걸기

1. 고객 카드 선택
2. 전화번호 탭
3. 자동으로 전화 앱 연결 → 통화 시작

### 4-6. 통화 기록 작성 (CallRecord)

통화 종료 후 반드시 결과를 기록해야 합니다.

1. 해당 고객 카드 → `통화 기록` 버튼
2. 결과 선택:

| 결과 | 설명 | 상태 변경 |
|------|------|---------|
| **성공** | 상담 동의 완료 | → SUCCESS |
| **거절** | 상담 거절 | → REJECT |
| **부재** | 전화를 안 받음 | → TRYING |
| **결번** | 없는 번호 | → INVALID |
| **콜백** | 나중에 다시 전화 요청 | → CALLBACK |

3. 추가 정보 입력 (선택):
   - `관심 상품`: 고객이 관심 보인 상품명
   - `메모`: 통화 내용 요약
   - `감도`: HIGH(높음) / MID(보통) / LOW(낮음)
   - `콜백 예정일`: 콜백 선택 시 날짜/시간 지정

4. `저장` 버튼 → 기록 완료

> **성공 기록 시 녹취 업로드 안내**
> 성공 통화는 녹취 파일을 첨부하여 업로드할 수 있습니다.

### 4-7. 배정 추가 신청 (`배정 신청` 탭)

배정된 고객을 모두 처리했거나 더 많은 고객이 필요한 경우:

1. `배정 신청` 탭 선택
2. 원하는 추가 배정 수량 입력
3. `신청` 버튼 탭
4. 관리자 승인 후 자동 배정됨

> 승인 여부는 `배정 신청` 탭에서 상태 확인 가능
> (PENDING: 대기중 / APPROVED: 승인 / REJECTED: 거절)

### 4-8. 내 작업 현황 (`통계` 탭)

오늘의 작업 현황을 실시간으로 확인:

| 항목 | 설명 |
|------|------|
| 오늘 총 콜 수 | 오늘 통화 기록 작성 건수 |
| 성공 건수 | 오늘 성공 처리 건수 |
| 성공률 | 성공/총콜 × 100 |
| 거절 건수 | 오늘 거절 건수 |
| 잔여 배정 수 | 아직 처리하지 않은 고객 수 |

### 4-9. 공지사항 확인

- 메인 화면 또는 `공지` 탭에서 관리자 공지사항 확인
- 중요 공지는 빨간 표시 또는 상단 고정

### 4-10. 상담원이 지켜야 할 사항

1. **통화 후 즉시 기록**: 기록하지 않으면 정산에 반영되지 않습니다.
2. **정확한 결과 선택**: 잘못된 결과 선택은 재활용 데이터 품질에 영향을 줍니다.
3. **성공 건 녹취 첨부**: 관리자가 확인할 수 있도록 녹취 파일을 업로드하세요.
4. **콜백 메모**: 콜백 예정 고객은 반드시 날짜/내용을 메모에 남기세요.

## 5. 주요 용어 정리

| 용어 | 설명 |
|------|------|
| **TM** | Telemarketing, 전화 상담 영업 |
| **1차 TM** | 첫 번째 전화 상담 (SALES_TM 팀) |
| **2차 영업** | 1차 성공 후 계약/구매 유도 (SALES_MAIN 팀) |
| **배정 (Assignment)** | 상담원에게 고객을 할당하는 행위 |
| **재활용** | 거절/결번 고객을 세탁 후 다시 배정 |
| **세탁 (Recycle)** | 재활용 횟수 (recycle_count) |
| **감도 (Sentiment)** | 고객의 구매 의향 수준 (HIGH/MID/LOW) |
| **풀리퀘스트 (PullRequest)** | 상담원이 추가 배정을 요청하는 행위 |
| **사번 (Code)** | 상담원 식별 코드 (예: TM-ABC123, 자동생성) |
| **콜백 (Callback)** | 고객 요청으로 다시 전화하기로 한 건 |
| **결번 (INVALID)** | 없거나 잘못된 전화번호 |
| **정산** | 통화 결과에 따른 급여/수수료 계산 |
